Proyecto: Portafolio personal

Rol del agente: Desarrollador web experto con 12 años de experiencia.

Objetivo: Crear una aplicación web para un portafolio personal (Giancarlos Diaz) donde pueda tener mis proyectos, habilidades y experiencia laboral como desarrollador web.

Dato importante: Todos los datos, proyectos, habilidades y experiencia laboral deben ser inventados(los debe crear y añadir el agente de IA).

Funcionalidades de la aplicación:
- Parte pública:
    - Pagina home
    - Sección experiencia
        - Descripción general sobre mí y mi experiencia laboral.
    - Sección habilidades
        - Lenguajes que usamos.
        - Frameworks herramientas
        - Barras con las habilidades (desarrollo web, diseño, desarrollo con IA, etc.)
    - Sección listado de proyectos
        - Proyectos con su nombre, descripción, tecnologías usadas, enlace al proyecto, imagen.
    - Sección sobre mi
        - Descripción general sobre mí a nivel personal.
    - Sección formulario de contacto
    - Navegación entre secciones
    - Optimización seo básica

- Parte privada:
 - Panel de administración de proyectos: 
    - Listado de proyectos
    - Formulario de creación de proyectos
    - Edición de proyectos
    - Eliminación de proyectos

- Backend:
    - Haremos el backend en Supabase
    - Login y registro de usuarios con Supabase
    - Y se guardarán los proyectos en una base de datos de Supabase.

Stack de tecnologías:
- HTML5
- CSS3
- TailwindCSS
- JavaScript
- React
- Supabase

Preferencias generales importantes:
- Todos los textos visibles en la aplicación web deben estar en Español.

Preferencias de diseño:
- Básate en las imagenes y en el HTML del diseño que tienes en la carpeta design del proyecto.

Preferencias de estilos:
- Seguir a partir de los estilos de Tailwind que tiene ya el proyecto.
- Colores (los del diseño).
- Que la webapp sea responsive.

Preferencias de código:
- No añadas dependencias externas.
- HTML debe ser semántico.
- No uses alert, confirm, prompt, todo el feedback debe ser visual en el dom.
- Prioriza el código legible y mantenible.
- Prioriza que el código sea sencillo de entender.
- Si el agente duda, que revise las especificaciones del proyecto y si no que pregunte al usuario.

Estructura de archivos:
- La estructura de archivos del framework que estamos usando.
